# 메타신호 기반 데이터 수집·추정 사업 정리

---

## 1. 문제 인식 (현 상황)

* 글로벌 검색엔진·소셜·광고 플랫폼은 **공식 API 외 자동 수집을 거의 전면 차단**
* CAPTCHA / Cloudflare / 계정 탐지로 **비정상적 크롤링은 장기 운영 불가**
* 단순 스케줄러 + 크롤링 구조는 서비스 확장 시 관리비·리스크 급증

결론:

> **원본 데이터를 직접 긁는 모델은 구조적으로 지속 불가능**

---

## 2. 핵심 해법 개념

### 2.1 메타신호(Meta Signal)

* 콘텐츠 내용 ❌
* **콘텐츠 주변에서 발생하는 ‘행동 흔적’ ⭕**

예시:

* 리뷰 증가량
* 랭킹 변화
* 검색량 지수 변화
* 광고 계정/캠페인 증가
* 크리에이티브 변형 수
* 찜·문의·Q&A 증가

→ **무엇이 써졌는지가 아니라, 얼마나·얼마나 빠르게 움직이는가**

---

### 2.2 외부조합(External Combination)

* 단일 플랫폼 ❌
* **여러 열린 출처의 메타신호를 동시에 관측 ⭕**

조합 예시:

* 쇼핑 플랫폼 내부 신호 (리뷰·랭킹·가격)
* 검색 트렌드
* 뉴스/RSS
* 앱스토어 순위
* 커뮤니티 언급 빈도

→ 각 신호는 약하지만 **동시에 같은 방향이면 신뢰도 급상승**

---

## 3. 광고·매출 추정에 대한 현실적 관점

### 3.1 핵심 전제

* 광고비 ≠ 효과 ≠ 외부 반응
* **선형 비례 관계 아님 (왜곡 함수)**

왜 비례하지 않는가:

* 크리에이티브 품질
* 타이밍
* 한계효용 체감
* 채널 믹스

따라서 목표는:

> ❌ 광고비 정확 추정
> ⭕ **광고 ‘상태(State)’ 판별**

---

### 3.2 광고 상태 모델

* Test (소액 테스트)
* Scale (확대)
* Push (공격 집행)
* Peak (피크)
* Cooldown (감속)

출력 형태:

* Low / Mid / High
* 전주 대비 증감률
* 동카테고리 상대 위치

---

## 4. 듀얼 테스트(접촉 테스트)의 위치

### 4.1 결론

* 대량 노출·봇 접촉 ❌
* **소량·고정 기준점 보정용 듀얼 테스트 ⭕**

비중:

* 메타신호 90%
* 듀얼 접촉 테스트 10% (보정·검증용)

---

### 4.2 안전한 듀얼 테스트 방식

1. 저빈도 고정 환경 접촉

* 동일 계정/지역/시간
* 방향성만 확인

2. 공개 지표와 교차 검증

* 광고 라이브러리
* 신규 크리에이티브 등장

3. 사용자 제공 라벨

* 광고주가 제공한 상태 정보(Test/Scale 등)

---

## 5. 스마트스토어 매출 추정 구조

### 5.1 기본 공식 (개념)

* 매출 ≈ 판매량 신호 × 가격 × 보정계수

### 5.2 사용되는 메타신호

* 리뷰 증가량 (가장 강력)
* 가격/할인
* 랭킹 변화
* 찜/문의/Q&A
* 외부 검색·광고 신호

출력 원칙:

* 절대 매출 ❌
* 매출 구간 / 증가율 / 순위 ⭕

---

## 6. 국내에서 실제 운영 중인 유사 사업들 (공통점)

공통 구조:

```
공개 정보 수집
→ 변화량/증가율 계산
→ 카테고리 평균으로 보정
→ 구간·순위·상태 출력
```

특징:

* 실매출 직접 제공 ❌
* 추정·지표·트렌드 중심 ⭕
* 메타신호 조합 모델

---

## 7. 법적·운영 원칙 (절대선)

금지:

* 로그인 우회
* CAPTCHA 해제
* 비공개 API 접근
* 원문 대량 저장

허용:

* 공개 페이지 숫자
* 카운트/증가율/랭킹
* 원본 비저장 메타 데이터

---

## 8. n8n / 시스템 운영 관점

* 단순 반복 실행 → 스케줄러 충분
* 복수 단계·조건·실패 관리 → **n8n은 오케스트레이션 도구**

권장 구조:

* 크롤링 로직: Python/Playwright 유지
* n8n 역할:

  * 스케줄
  * 재시도
  * 분기
  * 로그/알림

---

## 9. 사업 방향 요약

핵심 문장:

> **"원본을 긁는 서비스는 망하고,
> 메타신호로 상태를 읽는 서비스만 살아남는다."**

이 모델의 용도:

* 전략 판단
* 타이밍 감지
* 경쟁사/시장 온도 파악

비용·리스크 대비:

* 장기 운영 가능
* 차단 가능성 낮음
* 확장성 높음

---

## 10. 최종 정리 한 줄

**메타신호 + 외부조합 + 최소 듀얼 보정 =
차단되지 않는 데이터 추정 사업의 현재 정답 구조**

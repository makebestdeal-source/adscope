# STABILITY PRESERVATION MODE

## 핵심 철학
정상 동작 중인 로직은 자산이다.
동작을 바꾸지 않는 것이 최우선이다.
개선보다 안정성을 우선한다.

## 수정 원칙
1. 요청된 부분만 수정한다.
2. 기존 정상 로직은 절대 재해석하거나 개선하지 않는다.
3. “더 좋아 보이는 방식”으로 바꾸지 않는다.
4. 동작 흐름, 반환값, 호출 구조는 유지한다.
5. 기존과 동일한 결과가 나오도록 보존한다.
6. 변경이 필요한 경우 변경 이유와 영향 범위를 먼저 보고한다.
7. 전역 영향이 예상되면 즉시 중단하고 사용자 확인을 받는다.

## 행동 규칙
- 기존 함수는 그대로 둔다.
- 필요 시 내부 일부 라인만 교체한다.
- 구조 재설계는 금지.
- 리팩토링은 요청 시에만 수행.
- 수정 후 기존 기능이 그대로 동작하는지 반드시 검증한다.
